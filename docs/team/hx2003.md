# HX2003's Project Portfolio Page
## Project: CardCollector
CardCollector is a lightweight command-line application for trading card enthusiasts to manage their collections quickly and efficiently.

### Enhancements
- **New Feature**: Added the ability to view a sorted list of cards
  - What it does: allows the user to view a list of cards sorted by various criteria,
  such as index, name, quantity, price, etc.
  - Justification: improves the product significantly because it makes it convenient for user
  to quickly view cards with the most desired attributes.
  - Relevant PRS: [#98](https://github.com/AY2526S2-CS2113-T11-3/tp/pull/98)

- **New Feature**: Added the ability to view a historical audit log to changes made to cards
  - What it does: whenever a card is added, modified, or removed, the changes are appended to the history.
  - Justification: improves the product significantly because the user may want to see past records.
  For example, when a card has been removed from the list, the user might want to see the price it was.
  Or when the user added note in the past, and wants to revisit it.
  - Highlights: this enhancement heavily interacts existing commands which modify the state of the cards,
  care was taken to ensure that the history was corrected recorded.
  - Relevant PRS: [#28](https://github.com/AY2526S2-CS2113-T11-3/tp/pull/28), [#71](https://github.com/AY2526S2-CS2113-T11-3/tp/pull/71)
- **Parser: Split Tokenizer**: 
  - What it does: splits an input string using regex expression as delimiter.
  - Justification: several commands in the parser uses whitespace delimited method for parsing.
  - Relevant PRS: [#71](https://github.com/AY2526S2-CS2113-T11-3/tp/pull/71)
- **Parser: Disambiguator**:
  - What it does: matches an input string against a list of keywords, to see which matches. Fuzzy partial matching is adopted based on the prefix.
  - Justification: sometimes user is lazy to type out full arguments, so its faster to type a part of the argument.
  - Relevant PRS: [#95](https://github.com/AY2526S2-CS2113-T11-3/tp/pull/95)
- **Parser: Exceptions**: Added 3 fine-grained exceptions for the `Parser`.
  - What it does: `ParseBlankCommandException`, `ParseUnknownCommandException`, and `ParseInvalidArgumentException`
    indicate various types of scenarios in which parsing fails. Appropriate feedback is given to the user depending on which issue it is.
  - Justification: during parsing, users may occasionally enter invalid inputs.
    These exceptions ensure traceability, providing users with clear context about the issue.
  - Relevant PRs: [#38](https://github.com/AY2526S2-CS2113-T11-3/tp/pull/38), [#58](https://github.com/AY2526S2-CS2113-T11-3/tp/pull/58)
- **Utilities: Box**:
  - What it does: a container that holds any value or `null`.
  - Justification: for example, a method can accept a `Box<T>` type, if `Box.of(null)` was provided, it means "explicitly set this field to null", whereas
    if `null` was provided, it means "do not change this field".
  - Relevant PRS: [#199](https://github.com/AY2526S2-CS2113-T11-3/tp/pull/199)
